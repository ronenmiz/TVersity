- (BOOL) playURI: (NSString *)  uri

block: (TVersityAFCommandBlock)  block
posBlock: (TVersityAFPositionInfoBlock)  posBlock
error: (NSError **)  outError 

Send a uri to play. This method returns the result of the command asynchronously.

Parameters
[in]uriThe uri to play
[in](optional)title The title of the media to play. Can be nil.
[in](optional)mimeType Mime type of the media to play. Can be nil.
[in]blockA block to be called asynchronously with the result of the command. The block is called in the main thread of the application.
[in](optional)posBlock A block to be called asynchronously during playback with position information. Can be nil. The block is called in the main thread of the application.
[out]outErrorOn return, if an error accured, outError will contain an error object with error information.

This method can return one of the following error codes: TVersityAFInvalidDeviceError No device is known with the specified id TVersityAFDeviceOfflineError The device is offline and cannot be accessed TVersityAFInvalidCommandError The device does not support this command or its current state does not allow it TVersityAFInvalidParamError The uri parameter is not valid, only HTTP urls are acceptable TVersityAFUnknownError

Remarks
The mimeType parameter is optional. If nil is used, the player will try to derive the mime type from the URI's extension.
Supported protocols for media URI are http and ios. ios protocol is internal to this SDK. It supports passing of urls extracted from iOS resources. For example a url can be extracted from an MPMediaItem calling the method valueForProperty with key MPMediaItemPropertyAssetURL. Another example is extracting the urls from the dictionary returned by calling the ALAsset method valueForProperty with key ALAssetPropertyURLs. An ios protocol uri looks like this "ios://<ios asset url>". For example an actual uri can look like that: "ios://ipod-library://item/item.mp3?id=2495898145950574024". You should always get the asset url using iOS APIs as explained above, rather than trying to compose it yourself. This will protect you from internal iOS API changes in the assets url format.
Returns
YES if the method succeeds. NO if an error accured.
