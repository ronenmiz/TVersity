- (BOOL) browseChildrenWithBlock: (TVersityAFBrowseChildrenBlock)  block
ofObject: (TVersityAFObject *)  object
startIndex: (NSUInteger)  startIndex
fetchCount: (NSUInteger)  fetchCount
error: (NSError **)  outError 

Get the children of one of the child objects of this server. Objects can be containers or items. The block is called asynchronously in the main thread of the application.

Parameters
[in]blockThe block to invoke with the children information. The block is called asynchronously in the main thread of the application.
[in]ofObjectThe child object of this server of which we want to get its children. If nil is passed, returns the children of the root of the server.
[in]startIndex0 based index from where to start the children fetch
[in]fetchCountMaximum number of children to fetch. Use 0 to fetch all children
[out]outErrorOn return, if an error accured, outError will contain an error object with error information.

This method can return one of the following error codes: TVersityAFInvalidDeviceError No device is known with the specified id TVersityAFDeviceOfflineError The device is offline and cannot be accessed TVersityAFInvalidCommandError The device does not support this command or its current state does not allow it TVersityAFUnknownError

Returns
YES if the method succeeds. NO if an error accured.

Device current status. Notice this value does not update dynamically. It represents the status of the device at the moment this instance of the object was created.
