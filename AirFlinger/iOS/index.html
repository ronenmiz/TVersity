TVersity AirFlinger iOS SDK Documentation

The TVersity AirFlinger SDK provides capabilities for streaming media between different devices on a network.
Using this SDK you can detect media renderer devices (players) and media server devices.
Then you can send media over the network (Fling) from different sources to the player devices, or query media servers for media content.

This Objective-C SDK is designed for developing applications for the iOS operating system.
It is thread safe, but is designed for being used from a GUI application without the need to handle threads or network related issues.
Therefore, whenever a method of this SDK should return an asynchronous response, it will be sent in the main thread of the application.

To start using the SDK, begin with the TVersityAFAirFlinger class.
Using this class you initialize the SDK and get a collection of media devices identified on the network.
Once you have the collection of devices, you can control them or query them using the subclasses of the TVersityAFDevice.
With a TVersityAFPlayer class, you can get information and control playback of media on a media renderer device.
With a TVeristyAFServer class, you can get information and browse the media available on a media server.

Other capabilites provided by this SDK are in helper classes which extend the iOS to provide easier access to media items, or add to existing iOS components features related to "media Flinging".
Check the Helpers modules for more details.

A good way to continue from here is to check the Modules page, and examine the demo application provided with the SDK.
